    P.s.: Необходимость ревью можно задавать полем при создании задачи, тогда статус Review не будет виден в процессе работы, если необходимость ревью не указана.

    Необходимость внесения изменений в документацию можно также задавать полем, чтобы после выполнения задачи автоматически создавался сабтаск на документацию;



    Вид тикета после выполнения задачи:


    Для реализации всего вышеперечисленного нам понадобятся:

    • REST API  Jira; (документация по REST API)
      • Создание тикетов;
      • Модификация свойств тикетов;
      • Написание комментариев;
      • Получение списков тикетов (при помощи JQL);
    • Webhooks; (документация по Webhooks)
      • Вызов внешних callback-функций на события в Jira;
    • ScriptRunner (документация по ScriptRunner)
      • Запуск кастомных скриптов в среде Jira (создание своих кнопок, интеграция с Git, все такое);


    Бизнес-процесс:

    • Open/Backlog; (Сделать / Беклог)
    • In Progress; (Выполнение)
    • On Review; (На ревью)
    • Resovled - на проверку (беклог QA инженеров);
    • In Shot / In Shot - OK - проверка задачи изолированно (тестирование) / протестирована изолированно;
    • In Build/ In Build - OK;
    • On Production / On Production - OK;
    • Closed.

    У нас есть только мануальное тестирование, поэтому In Shot, In Build и On Production временно убираем (тестирования задач изолированно у нас нет, на билде тоже нет, продакшена у нас тоже нет)

     пока что выкидываются, получается примерно так





    Дополнительные статусы, в которых может находится тикет:

    • Requirements — когда не получается оперативно получить от заказчика необходимые уточнения по задаче, а без них дальнейшая работа по тикету невозможна, тикет переводится в этот статус и назначается тому, кто должен дать разъяснения;
    • Suspended — если работа по тикету приостановлена, например, если разработчик заблокирован задачами смежной команды или был вынужден переключиться на более срочную задачу;
    • Reopened — задача может быть переоткрыта на разработчика после ревью, после тестирования, после неудачной попытки слияния ветки с master
